# 历历在目（MUM / makeUrmove）

一款面向K12学生的历史与文学理解学习工具。

用户上传叙事文本，AI自动拆解为多阵营卡牌，用户通过按历史顺序出牌的方式验证自己对文本逻辑的理解。


## 项目状态

概念验证阶段。核心机制和功能规格已完成定义，正在准备原型开发。当前文档体系已覆盖产品愿景、用户画像、竞品分析、功能清单、用例清单和模型解析极限报告。


## 名称含义

历历在目，指过去的事物清晰地呈现在眼前。产品取此名，意在让历史文本中的人物、事件和因果链条通过卡牌对局的方式清晰呈现——用户不是在抽象地阅读文字，而是在具体地看到每一个事件节点、每一张手牌、每一次出牌判定，让历史的脉络历历可见。


## 核心机制

用户上传一段历史或文学文本。大模型解析文本，识别出多个阵营（如安史之乱中的唐廷、安史叛军、地方观望势力），并为每个阵营生成对应的事件牌序列。用户选择其中一个阵营扮演，按历史事件发生的顺序依次出牌。

出牌正确则推进进度。出牌错误时，系统根据错误类型扣减生命值——完全错误扣一颗完整心，部分正确（没选全）扣半颗心。生命值归零时，用户可选择继续走完全程（不再计心）或放弃对局。

对局结束后，系统展示结算报告，呈现正确出牌数、错误次数、错误发生的具体历史事件位置，以及综合评级。用户可根据错误定位回到原文重新阅读，针对性弥补理解盲区。

AI在本产品中不充当对手。所有AI阵营按固定剧本同步推进历史事件，其作用是让用户看到其他阵营在同一时间节点发生了什么，形成完整的历史图景。胜负不是看用户是否赢了AI，而是看用户是否正确复现了自己阵营的历史路径。


## 差异化定位

与背诵类工具（如百词斩、背书匠）不同，历历在目聚焦于叙事文本的整体逻辑理解，而非零散知识点的记忆。用户不是在背诵单个事件，而是在验证自己对事件因果链条的理解是否正确。

与策略卡牌游戏不同，历历在目的胜负服务于认知目标。结算报告中的错误位置精确到具体历史事件，帮助用户定位理解盲区并针对性复习。所有游戏化机制（出牌、扣血、评级）都服务于理解检验这一核心目标。

用户上传任意文本即可生成对局，内容动态生成而非依赖固定题库。同一段文本在不同参数下可生成不同的卡牌分配，增加文本的可重玩性。


## 路线图

第一阶段：完成历史顺序复现模式的核心闭环。实现文本解析、阵营与事件牌生成、出牌判定、生命值系统、AI同步出牌和结算报告。目标是验证核心学习机制的有效性。

第二阶段：启动历史走向模拟（What if）模式的开发。用户在关键节点可选择替代历史选项，系统推演该选择可能带来的结果变化，帮助用户理解历史进程中的因果依赖性。

第三阶段：扩展预设文本库（与教材标准内容对齐），支持多人参与模式，实现知识图谱联动。
